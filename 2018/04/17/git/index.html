git的常用命令

Posted on 2018-04-17

学习了一些，git常用命令，整理一下，方便平时参考

(基础知识：cd前往某文件，pwd显示全路径，mkdir创建新文件夹)

创建:

添加文件到Git仓库，分两步：

注意可以使用多次，添加多个文件；然后add到暂存区中，等待commit到分支中
第二步，使用命令 Git commit，完成，以commit到分支中，工作区clean
如果不先将文件add在暂存区中，是commit不了的

版本回退：

Git允许我们在版本的历史之间穿梭,用HEAD表示当前版本，上一个版本就是HEAD^，上上一个版本就是HEAD^^，当然往上100个版本写100个^比较容易数不过来，所以写成HEAD~100

checkout：

撤销在工作区的修改，无论是未add在暂存区的，还是已add后再在工作区修改的

push:

github.com 与本地Git的连接。
首先使用ssh-keygen -t rsa -C “youremail@example.com“
然后确认，在输入y，直接一路回车。
使用命令：git remote add origin git@github.com:Tec-yao/learngit.git
成功关联后，使用Git push -u origin master.推送，以后再往该仓库中push，就可以直接Git push origin master
假如出现fatal:remote rm origin 解决办法如下
先输入：git remote rm origin
在输入：git remote add origin git@github.com:Tec-yao/learngit.git

clone:

git clone git@github.com:Tec-yao/gitskills.git
如果出现以下问题，则检查pubkey有没添加到GitHub上，
或者使用管理员模式
Cloning into ‘gitskills’…
git@github.com: Permission denied (publickey).
fatal: Could not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.

分支管理：

创建与合并

解决冲突：

当两分支修改同一出，然后合并时，将出现冲突。
修改，手动修改重新，然后add commit

bug修复：

当你代码出现bug时，可以使用Git stash 保存当下branch。然后创建新branch去修改，然后合并修改完代码读出原branch,两种，一是git stash apply恢复，但是stash内容不删除，需要使用git stash drop 删除。另一是git stash pop 恢复的同时把stash的内容也删除。可以随时使用git stash list查看stash内容。需要开发新功能，最好建立新的分支。删除一个未合并的分支使用git branch -D 强行删除

本地推送：

打标签先切换到需要打tag的分支，然后打标签。默认最新提交commit
需要打历史commit的tag就是使用Git log –pretty=onrline –addrev-commit 找到commit id.打上就好吗，格式如：git tag tag-name commit-id前7位

其他：

使用Github参与开源项目，先访问主页，点击fork在自己的账户上clone一个仓库，然后在自己的账号上clone到本地，在本地开发。最后想让官方库接受你的修改，就得pull request.

在git工作区创建特殊的.gitignore,将需要忽略的文件名填入。如保存数据库密码的配置文件。
