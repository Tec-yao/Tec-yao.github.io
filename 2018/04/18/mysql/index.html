约束

约束保证数据的完整性和一致性
约束分为表级约束和列级约束。
如果只约束一个字段则为列级约束，如果是约束多个字段为表级约束
总的来说，约束就是在创建表的时候，在字段后添加一些限制条件，以达到一些目的

主键约束

一个表只能存在一个主键，主键保证记录的唯一性，自动为NULL。
其中主键可配合AUTO_INRCREMENT,会自动编号，无需用户主动添加。

CREATE TABLE tb(
id SMALLINT UNSIGNED PRIMARY KEY [AUTO_INCREMENT],
username VARCHAR(30);
)

唯一约束

添加UNIQUE KEY

默认约束

当插入记录时，如果没有明确为字段赋值，则自动赋予默认值

非空约束

也可以添加空与非空，在插入表时在变量后添加NOT NULL，即为非空，不可不填。
如果为NULL，可为空，默认为NULL。

外键约束

1.父表和子表必须使用相同的储存引擎，而且禁止使用临时表。
2.数据表的储存引擎只能为INNODB.
3.外键列的参照列必须具有相似的数据类型。其中数字的长度
或是否符号位必须相同；而字符的长度则可以不同。
4.外键列和参照列必须创建索引。如果外键列不存在索引的话，MySQL将自动创建索引。

外键约束的参照操作

CASCADE：从父表删除或自动更新且自动删除或更新子表中匹配的行
SET NULL: 从父表删除或更新行，并设置子表中的外键列为NULL。
如果使用该选项，必须保证子表列没有指定NOT NULL
RESTRICT: 拒绝对父表的删除或更新操作
NO ACTION: 标准SQL的关键字，在MySQL中与RESTRICT相同

查询表达式：
每一个表达式不是想要的一列，必须有至少一个。
多个列之间以英文逗号分隔。
星号()表示所有列。tbl_name.可以表示命名表的所有列。
查询表达式可以使用[AS] alias_name 为其赋予别名。
别名可用于GROUP BY, ORDRE BY或HAVING字句。

其中：查询列时，可改变顺序查询，也可使用AS别名查询。

条件表达式WHERE

对记录进行过滤，如果没有指定WHERE字句，则显示所有记录。
在WHERE表达式中，可以使用MySQL支持的函数或运算符。

查询结果分组 GROUP BY

对于查询内容用列名或者位置分组

分组条件 HAVINIG
[HAVING where_condition]
其中需要注意的是如：
SELECT sex,age FROM users3 GROUP BY 1 HAVING age > 20;
SELECT sex,age FROM users3 GROUP BY 1 HAVING count(id);
要么条件中列名要再查找的列名中如1，
要么条件中使用聚合函数。

对查询结果进行排序：

[ORDER BY {col_name | expr | position} [ASC|DESC],…]

可以只用一个列名进行排序，也可以多个进行排序。用逗号分开即可。

限制查询结果返回LIMIT

[LIMIT {[offer,] row_count | row_count OFFSET offset}]

LIMIT 返回几条结果；
LIMIT 从第几条开始（0开头），返回几条结果。

存储过程

存储过程是SQL语句和控制语句的预编译集合，以一个名称存储并作为一个单元处理

优点：
增强SQL语句的功能和灵活性
实现较快的执行速度
减少网路流量

创建存储过程

CREATE
[DEFINER = {user|CURRENT_USER}]
PROCEDURE sp_name ([proc_parameter[,…]])
[characteristic …] routine_body

proc_parameter:
[IN|OUT|INOUT] param_name type

参数：
IN 表示该参数的值必须在调用存储过程时指定
OUT 表示该参数的值可以被存储过程改变，并且可以返回
INOUT 表示该参数的调用时指定，并且可以被改变和返回

无参存储过程：

调用时用CALL 有无括号都行

使用IN OUT的存储过程
CREATE PROCEDURE removeUserAndrReturnNums(IN p_id INT UNSIGNEDM,OUT userNums INT UNSIGNED)
BEGIN
DELETE FROM users WHERE id = p_id;
SELECT count(id) FROM users INTO userNums;
END

使用：CALL removeUserAndReturnNums(27,@nums);

SELECT @nums;

存储过程与自定义函数的区别
存储过程实现的功能要复杂一些，函数针对性强
存储过程可以返回多个值，函数只能有一个返回值
存储过程一般独立的来执行，函数可以作为其他SQL函数的组成部分出现

总结：
SQL语句和控制语句的预编译集合，以一个名称作为一个单元处理。
参数为输入，输出，亦可以并存。
CREAT … PROCEDURE

注意事项：
1.创建时用DELIMITER修改界定符。
2.如果有多条语句，需要包含在BEGIN..END
3.用CALL调用

存储引擎

MySQL可以将数据以不同 的技术存储在文件中，这种技术就称为存储引擎、
每一种存储引擎使用不同的存储机制。索引技巧。锁定水平，最终提供广泛且不同的功能

存储引擎：
1.MyISAM
2.InnoDB
3.Memory
4.CSV
5.Archive

并发控制。

当多个连接对记录进行修改是保证数据的一致性和完整性
使用锁系统来控制

分为共享锁(读锁)：
在同一时间段内多个用户可以读取同一资源。读取过程中数据不会发生任何变化。
排他锁(写锁) ：
在任何时候只能有一个用户写入资源，当进行写锁是会阻塞其他的读锁或写锁操作

表策略：
表锁，是一种开销最小的锁策略
行锁，是一种开销最大的锁策略

事务处理

事务用于保证数据库的完整性
将几个过程作为一个整体，不论哪一个环节出了问题都需要恢复至原始的数据
满足四个性质
原子性
一致性
隔离性
持久性

索引

是对数据表中一列或者多列的值进行排序的一种结构
