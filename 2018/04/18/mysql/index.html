约束

约束保证数据的完整性和一致性
约束分为表级约束和列级约束。
如果只约束一个字段则为列级约束，如果是约束多个字段为表级约束
总的来说，约束就是在创建表的时候，在字段后添加一些限制条件，以达到一些目的

主键约束

一个表只能存在一个主键，主键保证记录的唯一性，自动为NULL。
其中主键可配合AUTO_INRCREMENT,会自动编号，无需用户主动添加。

CREATE TABLE tb(
id SMALLINT UNSIGNED PRIMARY KEY [AUTO_INCREMENT],
username VARCHAR(30);
)

添加UNIQUE KEY

默认约束

当插入记录时，如果没有明确为字段赋值，则自动赋予默认值

非空约束

也可以添加空与非空，在插入表时在变量后添加NOT NULL，即为非空，不可不填。
如果为NULL，可为空，默认为NULL。

外键约束

1.父表和子表必须使用相同的储存引擎，而且禁止使用临时表。
2.数据表的储存引擎只能为INNODB.
3.外键列的参照列必须具有相似的数据类型。其中数字的长度
或是否符号位必须相同；而字符的长度则可以不同。
4.外键列和参照列必须创建索引。如果外键列不存在索引的话，MySQL将自动创建索引。

外键约束的参照操作

CASCADE：从父表删除或自动更新且自动删除或更新子表中匹配的行
SET NULL: 从父表删除或更新行，并设置子表中的外键列为NULL。
如果使用该选项，必须保证子表列没有指定NOT NULL
RESTRICT: 拒绝对父表的删除或更新操作
NO ACTION: 标准SQL的关键字，在MySQL中与RESTRICT相同
